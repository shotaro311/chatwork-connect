# chatwork-connect

チャットワークをAIエージェント（Claude Code / Codex）から読み書きできるようにするパッケージです。
画面の自動操作は使わず、チャットワーク公式の窓口（API）経由で動きます。

## できること

- 未読メッセージ・自分宛メンションの確認
- ルームごとのメッセージの読み取り
- 返信文の下書き作成 →「あなたの承認後にだけ」送信
- ルームの既読化
- 新着メッセージのローカル保存（導入した日から履歴を貯めていく）
- 過去ログの一括取り込み（**既定はオフ**。詳細は下記）

**勝手にメッセージを送らない設計です。** 送信は必ず、あなたが下書きを確認して「OK」と言ってから実行されます。

## 必要なもの

- チャットワークのアカウント（無料プランでも可）
- Claude Code または Codex
- Python 3.9 以上（追加のインストールは不要。Macは標準で入っています）

## 導入方法

**Claude Code（または Codex）に、下の文をそのまま貼り付けるだけです。**

```
https://github.com/shotaro311/chatwork-connect.git をこのフォルダにクローンして、
中の SETUP.md を読んでセットアップを進めてください。
```

あとはAIが順番に案内します。あなたの操作は次の3つだけです。

1. 案内されたリンクを開いて、APIトークンを**コピー**する
2. AIが出すコマンドの**実行ボタンを押す**（入力は不要です）
3. 完了報告を確認する

ファイルを開いたり、パスを打ち込んだりする必要はありません。

---

**Claude Code / Codex のプラグインとして入れる場合**（任意）

このリポジトリはプラグイン形式にも対応しています。

```
# Claude Code
/plugin marketplace add shotaro311/chatwork-connect

# Codex
codex plugin marketplace add https://github.com/shotaro311/chatwork-connect.git
```

## 使い方の例

セットアップ後は、AIにそのまま話しかけるだけです。

- 「チャットワークの未読を確認して」
- 「◯◯さんのルームの最新のやり取りを見せて」
- 「このメッセージに返信文を作って」→ 下書き確認 → 「OK、送って」

## 過去ログの一括取り込みについて（既定はオフ）

導入より前のメッセージをまとめて取り込む機能です。**安全のため、はじめはオフになっています。**

オンにする前に、以下をご確認ください。

- チャットワークの**公式ドキュメントに載っていない方法**を使うため、将来動かなくなる可能性があります
- 件数が多い場合、**数時間かかる**ことがあります（速度制限のため）
- **無料プランでは、40日より前のメッセージはこの機能でも取得できません**

```
# 現在の設定を見る
python3 scripts/chatwork.py config

# オンにする
python3 scripts/chatwork.py config --set history_import=on

# 取り込む（まずは少しだけ試すのがおすすめ）
python3 scripts/chatwork.py import-history --max-requests 20

# 直近30日ぶんだけ取り込む場合
python3 scripts/chatwork.py import-history --days 30

# 終わったらオフに戻す
python3 scripts/chatwork.py config --set history_import=off
```

途中で止まっても、同じコマンドを実行すれば**続きから再開**します。
AIエージェントに「過去ログを取り込んで」と伝えれば、この手順も案内してくれます。

## 注意事項

- **APIトークンは「合鍵」です。** 他人に見せない・チャットに貼らない・`.env` ファイル以外に書かない。
- **無料プランの制限**: チャットワーク無料プランでは、閲覧できるメッセージが「直近40日以内」に制限されており、APIでの取得も同じ制限を受けます。
- **過去ログの遡り**: 公式APIで取得できるのは各ルーム最新100件までです。それより前は上記の「一括取り込み」機能を使いますが、公式にサポートされた方法ではありません。
- 法人契約（ビジネス/エンタープライズプラン）の場合は、APIトークン発行前に組織管理者の承認が必要です。

## フォルダ構成

```
chatwork-connect/
├── README.md          ← このファイル（人間向け）
├── SETUP.md           ← AIエージェント向けのセットアップ手順書
├── .env.example       ← トークン設定ファイルの見本
├── scripts/
│   └── chatwork.py    ← チャットワークと通信するプログラム（Python3・追加インストール不要）
├── skills/
│   ├── chatwork-check/  ← 未読確認の手順書（Skill）
│   ├── chatwork-reply/  ← 返信作成〜承認〜送信の手順書（Skill）
│   └── chatwork-log/    ← 履歴のローカル蓄積の手順書（Skill）
└── config/
    └── reply-policy.example.md  ← 返信ルール（口調・承認条件）のひな型
```

トークンとログの保存先は `~/.config/chatwork-connect/`（パッケージを更新しても消えません）。
