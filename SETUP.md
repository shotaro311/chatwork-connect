# SETUP.md — AIエージェント向けセットアップ手順書

あなた（AIエージェント）は、このパッケージ `chatwork-connect` のセットアップを担当します。
**利用者は非エンジニアです。** ファイルを手で開かせる、パスを打たせる、といった操作は求めないでください。
利用者の操作は「**リンクをクリックする**」「**コピーする**」「**実行ボタンを押す**」の3つだけで完了させます。

---

## 絶対に守るルール

1. **APIトークンをあなたが受け取らない。** チャット欄に貼るよう求めない。クリップボードを読み取らない。
   ファイルに書き込むのもあなたではなく、**利用者が実行ボタンを押して行います**（Step 3）。
2. 利用者がチャット欄にトークンを貼ってしまった場合は、**その値を復唱・表示・保存しない**。
   「安全のため、いまのメッセージは削除して、次の手順で設定しましょう」と伝えて Step 3 へ進む。
3. **メッセージ送信（`send`）はセットアップ中に実行しない。** 接続テストは読み取りのみ。
4. **過去ログの一括取り込みを勝手にオンにしない。** 既定はオフのままにする。
5. 失敗したら Step 5 のトラブルシュートを参照し、解決しなければ状況を説明して指示を待つ。

---

## Step 0: 準備の確認

このファイルが読めていれば、パッケージは手元にあります。
無い場合は、作業フォルダで次を実行してから続けてください。

```
git clone https://github.com/shotaro311/chatwork-connect.git
```

次に、Pythonを確認します。

```
python3 --version
```

- Windowsで動かない場合は `python --version` を試す。3.9以上ならOK
- 以降のコマンドは、動いた方（`python3` / `python`）に読み替える
- どちらも無い場合は https://www.python.org/downloads/ を案内する

**この時点で利用者のOS（Mac / Windows）を必ず把握してください。** Step 3 で出すコマンドが変わります。

---

## Step 1: APIトークンの発行を案内する

利用者に、次のとおり伝えます（リンクはそのまま提示する）。

> チャットワークの「APIトークン」を発行します。下のリンクを開いてください。
>
> https://www.chatwork.com/service/packages/chatwork/subpackages/api/token.php
>
> パスワードの再入力を求められたら入力してください（正常な動作です）。
> 表示されたトークンを**コピー**したら教えてください。
> **コピーするだけでOKです。どこかに貼り付ける必要はありません。**

補足として伝えること:

- 法人プランの場合、「管理者の承認が必要」と表示されることがある。その場合は社内のチャットワーク管理者に承認を依頼する

利用者から「コピーした」と返ってきたら Step 2 へ進みます。

---

## Step 2: トークンを設定する ★ここが核心

**利用者がクリップボードにトークンをコピーした状態**で、OSに応じて下のコマンドを
**そのままコードブロックとして提示**します。利用者は実行ボタンを押すだけです。

### Mac の場合

提示するコードブロック（`bash` 指定）:

```bash
mkdir -p ~/.config/chatwork-connect && printf 'CHATWORK_API_TOKEN=%s\n' "$(pbpaste)" > ~/.config/chatwork-connect/.env && chmod 600 ~/.config/chatwork-connect/.env && echo "設定が完了しました"
```

### Windows の場合

提示するコードブロック（`powershell` 指定）:

```powershell
$t = (Get-Clipboard -Raw).Trim(); New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.config\chatwork-connect" | Out-Null; Set-Content -Path "$env:USERPROFILE\.config\chatwork-connect\.env" -Value "CHATWORK_API_TOKEN=$t" -Encoding ascii; Write-Host "設定が完了しました"
```

提示するときの言い方:

> コピーしていただいたトークンを、そのまま設定ファイルに保存します。
> 下のコマンドの**実行ボタンを押してください**（入力は不要です）。
> トークンは私からは見えない形で保存され、他の人には読めない権限が設定されます。

**重要**: このコマンドを**あなたが実行してはいけません。必ず利用者に実行してもらいます。**
クリップボードの中身を読むコマンドのため、実行するのは利用者本人である必要があります。

「設定が完了しました」と表示されたら Step 3 へ進みます。

---

## Step 3: 接続テスト

ここからはあなたが実行します。トークンの値は出力されないため、実行して問題ありません。

```
python3 scripts/chatwork.py me
```

利用者のアカウント名が返れば成功です。続けて状態を確認します。

```
python3 scripts/chatwork.py status
python3 scripts/chatwork.py rooms --unread-only
```

---

## Step 4: 完了報告

次の内容を、専門用語を使わずに報告します。

- 接続できたアカウント名
- 未読のあるルーム数（あれば）
- これからできること:
  - 「チャットワークの未読を確認して」
  - 「このメッセージに返信文を作って」← **送信前に必ず確認が入ることを明記する**
  - 「チャットワークのログを保存して」
- 補足:
  - 履歴は**今日から**保存されていくこと
  - 過去ログの一括取り込み機能もあるが**既定でオフ**であること。必要なら相談してほしいこと

---

## Step 5: トラブルシュート

| 症状 | 対処 |
| --- | --- |
| `認証エラー(401)` | トークンのコピーミス。Step 1 からやり直す（再発行も可） |
| `APIトークンが未設定です` | Step 2 のコマンドが実行されていない。実行ボタンを押したか確認する |
| Windowsで `Get-Clipboard` が使えない | コマンドプロンプトの可能性。PowerShellで実行するよう案内する |
| トークン発行画面で承認待ちになる | 法人プラン。社内の管理者へ承認を依頼するよう案内する |
| `利用回数制限(429)` | 5分待って再実行 |
| メッセージが0件しか取れない | 無料プランは直近40日以内のみ取得可能。仕様であることを説明する |
| `接続に失敗しました` | ネットワークまたはプロキシを確認する |
