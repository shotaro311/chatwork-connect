# SETUP.md — AIエージェント向けセットアップ手順書

あなた（AIエージェント）は、このパッケージ `chatwork-connect` のセットアップを担当します。
以下の手順を上から順に実行してください。ユーザーは非エンジニアの場合があります。専門用語を避け、丁寧に案内してください。

## 絶対に守るルール

1. **APIトークンを画面に表示・復唱・ログ出力しない。** ユーザーが貼り付けた後も、値を読み上げない。
2. **`.env` を Git にコミットしない。** このフォルダが Git 管理下にある場合、`.gitignore` に `.env` を追加する。
3. **メッセージ送信（`send`）はセットアップ中に一切実行しない。** 接続テストは読み取り系のみ。
4. 手順の途中で失敗したら、後述のトラブルシュートを参照し、解決しない場合は状況をユーザーに説明して指示を待つ。

## 手順

### Step 1: 実行環境の確認

Python 3 が使えるか確認する。

```
python3 --version
```

- 動かない場合は `python --version` を試す（Windows）。バージョン 3.9 以上であればOK。
- どちらも無い場合: ユーザーに Python のインストールが必要なことを伝え、公式サイト（https://www.python.org/downloads/）を案内する。以降のコマンドは環境に合わせて `python3` / `python` を読み替える。

### Step 2: トークン設定ファイルの準備

1. 設定フォルダを作成する: `~/.config/chatwork-connect/`
2. このパッケージの `.env.example` を `~/.config/chatwork-connect/.env` としてコピーする。

> トークンの置き場所は `~/.config/chatwork-connect/.env` を推奨する（プラグインの更新・再インストールで消えないため）。
> パッケージ直下に `.env` を置く方式も動作するが、その場合はパッケージフォルダが Git 管理下にあるか確認し（`git rev-parse --is-inside-work-tree`）、管理下なら `.gitignore` に `.env` を追記する。

### Step 3: APIトークンの取得案内（ユーザー操作）

ユーザーに以下を案内する:

> チャットワークの「合鍵」にあたるAPIトークンを取得します。
> 1. ブラウザでチャットワークを開く
> 2. 画面右上の自分のアイコン →「サービス連携」をクリック
> 3. 左メニューの「API」→「APIトークン」を開く（パスワードの再入力を求められることがあります）
> 4. 表示されたトークンをコピーする
>
> ※法人契約の場合、ここで「管理者の承認が必要」と表示されることがあります。その場合は組織の管理者に承認を依頼してください。

### Step 4: トークンの設定（ユーザー操作）

ユーザーに `~/.config/chatwork-connect/.env` を開いてもらい、`CHATWORK_API_TOKEN=` の右側にトークンを貼り付けてもらう。

- 可能であれば、エディタで `.env` を開いた状態にして「この行に貼り付けて保存してください」と案内する。
- ユーザーがチャット欄にトークンを直接貼ってきた場合: 受け取って `.env` に書き込んでよいが、**以降その値を一切表示しない**。書き込み後、「チャット欄からトークンを削除することをおすすめします」と伝える。

### Step 5: 接続テスト

```
python3 scripts/chatwork.py me
```

- 成功するとユーザーのアカウント名が表示される。「◯◯さんのアカウントで接続できました」と報告する。
- 続けて動作確認:

```
python3 scripts/chatwork.py status
python3 scripts/chatwork.py rooms --unread-only
```

### Step 6: 完了報告

以下の形式でユーザーに報告する:

- 接続できたアカウント名
- 未読のあるルーム数（あれば）
- 使い方の例: 「チャットワークの未読を確認して」「この人に返信文を作って」
- 注意: 送信は必ず承認後にのみ行うこと、無料プランでは直近40日以内のメッセージのみ取得できること

### Step 7（任意）: 返信ルールの設定

ユーザーに `config/reply-policy.example.md` の存在を伝え、返信の口調や「自動送信を許可する相手」を設定したい場合は `reply-policy.md` としてコピーして編集できることを案内する。今は設定しなくてもよい。

### Step 8（任意）: 履歴ログの蓄積開始

チャットワークAPIは過去ログを遡れない（各ルーム最新100件・無料プランは直近40日以内のみ）。
履歴を残したい場合は、今日から蓄積を始められることを案内する:

```
python3 scripts/chatwork.py archive
```

詳細は `skills/chatwork-log/SKILL.md` を参照。定期実行を希望されたらそのSkillの手順に従う。

## トラブルシュート

| 症状 | 原因と対処 |
| --- | --- |
| `認証エラー(401)` | トークンの貼り付けミスの可能性。前後の空白や欠けがないか、`.env` を確認してもらう |
| `利用回数制限(429)` | 5分あたり300回の制限。5分待って再実行 |
| トークン発行画面で承認待ちになる | 法人契約のため管理者承認が必要。管理者への依頼を案内 |
| `接続に失敗しました` | ネットワークまたはプロキシの問題。インターネット接続を確認 |
| メッセージが取得できない/少ない | 無料プランは直近40日以内のみ取得可能。仕様であることを説明 |
